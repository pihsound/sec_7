PART 01

허니팟 (Honeypot)

🍯
SECTION 1-1

허니팟이란?

공격자를 유인하기 위해 의도적으로 취약하게 구성된 함정 시스템

허니팟(Honeypot)은 사이버 보안에서 공격자를 유인하기 위해 의도적으로 취약점이 있거나 매력적으로 보이도록 설계된 가짜 시스템·자원이다. 꿀단지(honeypot)에 벌이 유인되듯, 공격자를 끌어들여 행동을 관찰·기록하고 실제 시스템을 보호한다.

💡 허니팟의 핵심 가치
허니팟에 접근하는 트래픽은 100% 비정상(악의적) 트래픽이다. 실제 사용자가 허니팟에 접근할 이유가 없기 때문이다. 따라서 허니팟 접근 = 즉각 경보 → 오탐(False Positive)이 거의 없는 탐지 방법이다.
⚙️
SECTION 1-2

허니팟 운용 흐름

유인 → 탐지 → 기록 → 분석 → 위협 인텔리전스화
1
🎣 유인 (Lure)
허니팟을 매력적인 표적으로 구성한다. 취약한 버전의 SSH·FTP 서버, 가짜 관리자 패스워드, 실제처럼 보이는 중요 파일명을 배치하여 공격자가 스캔 시 발견하도록 한다.
2
🪤 침투 허용 (Controlled Engagement)
공격자가 허니팟에 침투하도록 제한적으로 허용한다. 실제 시스템과 격리되어 있어 피해가 없으며, 공격자는 진짜 시스템에 침투했다고 착각한다.
3
📹 전체 기록 (Full Capture)
공격자의 모든 행동을 기록한다: 사용 명령어·도구·악성코드, 공격 IP와 시간, 탈취 시도한 데이터, 이동 경로, C&C 서버 주소 등.
4
🔬 분석 (Analysis)
수집된 공격 데이터를 분석하여 새로운 공격 기법·취약점·악성코드 패턴을 파악한다. 제로데이 취약점이나 미지 APT 기법을 발견하는 경우도 있다.
5
📡 위협 인텔리전스화 (Threat Intel)
분석 결과를 IoC(침해지표), 시그니처, 보안 패치로 변환하여 실제 방어 체계에 적용한다. 전 세계 허니팟 네트워크(예: Shodan, GreyNoise)의 데이터를 공유한다.
📁
SECTION 1-3

허니팟 유형 분류

상호작용 수준 · 목적 · 형태에 따른 분류
① 상호작용 수준에 따른 분류
저 인터랙션 허니팟
Low-Interaction Honeypot
에뮬레이션(모방)된 서비스만 제공한다. 실제 OS가 아닌 가짜 응답만 모방하여 공격자와 제한적으로 상호작용한다. 배포가 쉽고 안전하지만 수집 정보가 제한적이다.

예) Honeyd, Kippo(SSH 에뮬레이터)
안전 배포 쉬움 정보 제한적 탐지 가능성 있음
고 인터랙션 허니팟
High-Interaction Honeypot
실제 OS와 서비스를 운영하여 공격자가 완전히 상호작용할 수 있게 한다. 공격자가 진짜 시스템으로 오해하고 다양한 기법을 시도하므로 풍부한 정보 수집이 가능하다. 관리가 복잡하고 위험하다.

예) 가상머신 기반 실제 서버
풍부한 정보 실제와 동일 관리 복잡 탈출 위험 있음
② 목적에 따른 분류
유형목적특징
연구용 허니팟 새로운 공격 기법·악성코드 연구, 위협 인텔리전스 수집 학술·보안 연구 기관이 운용. 데이터 분석에 집중
운영용 허니팟 실제 조직의 내부 네트워크 보호, 침입 조기 탐지 기업·기관이 실제 방어용으로 배치. 경보 시스템과 연동
③ 형태에 따른 분류
형태설명예시
허니팟 (Honeypot) 단일 가짜 시스템 가짜 SSH 서버, 가짜 웹서버
허니넷 (Honeynet) 여러 허니팟으로 구성된 가짜 네트워크 가짜 기업 내부망 전체 구성
허니토큰 (Honeytoken) 가짜 자격증명·문서·API 키 등 데이터 함정 가짜 AWS API 키, 가짜 비밀번호 파일
허니파일 (Honeyfile) 접근 시 경보를 발생시키는 가짜 중요 파일 "비밀계획.docx", "passwords.txt"
허니유저 (Honeyuser) 실제 사용 안 하는 가짜 계정 (캐나리 계정) admin_backup, sa, root2 계정

SECTION 1-6

허니팟 장단점 & 법적 이슈

✓ 장점
  • 허니팟 접근 = 100% 악성 트래픽 → 오탐(FP) 거의 없음
  • 암호화된 공격도 탐지 가능 (내부 행동 관찰)
  • 새로운 공격 기법·제로데이 발견 가능
  • 공격자의 주의를 분산시켜 실제 시스템 보호
  • 저렴한 비용으로 운영 가능 (저 인터랙션의 경우)
  • 위협 인텔리전스 생성 기여
✗ 단점
  • 허니팟을 직접 공격하지 않는 공격은 탐지 불가
  • 고 인터랙션 허니팟은 관리·유지 비용 높음
  • 공격자가 허니팟임을 인식하면 회피
  • 허니팟을 거점으로 실제 시스템 공격 위험
  • 법적 문제: 공격자 유인이 '함정 수사'로 판단될 수 있음
⚖️ 허니팟 법적 이슈 — Entrapment(함정 수사)
허니팟은 적극적인 유인(Entrapment)이 아닌 수동적 관찰이어야 법적으로 허용된다. 공격자가 스스로 접근하도록 환경을 만드는 것은 허용되지만, 공격자를 적극적으로 유도하거나 실제 공격 행위를 부추기면 법적 문제가 발생할 수 있다. 허니팟 운영 전 법률 검토가 필수이다.
📝 기출 유형
확인 문제 — 허니팟
Q1 · 군무원 / 정보보호 기사 유형
허니팟(Honeypot)에 대한 설명으로 옳지 않은 것은?
①실제 서비스를 모방하여 공격자를 유인하고 행동을 기록·분석하는 시스템이다.
②허니팟에 접근하는 트래픽은 정상 사용자가 없으므로 대부분 악의적 트래픽이다.
③허니팟은 항상 실제 OS와 서비스를 그대로 운영하므로 배포 비용이 매우 높다.
④허니팟을 이용하여 새로운 공격 기법이나 악성코드를 수집할 수 있다.
✅ 정답: ③
저 인터랙션 허니팟(Low-Interaction)은 실제 OS가 아닌 에뮬레이션된 서비스만 제공하므로 배포가 쉽고 비용이 저렴하다. 실제 OS를 운영하는 것은 고 인터랙션 허니팟(High-Interaction)의 특징이다. 두 방식 모두 존재하며 목적에 따라 선택한다.
Q2 · 군무원 유형
허니팟의 형태 중 허니토큰(Honeytoken)에 해당하는 것은?
①여러 개의 허니팟으로 구성된 가짜 네트워크 전체
②저 인터랙션 방식으로 SSH 서비스를 에뮬레이션하는 시스템
③실제로 사용되지 않는 가짜 API 키, 자격증명, 문서 등의 데이터 함정
④공격자의 아웃바운드 트래픽을 제어하는 허니넷의 게이트웨이
✅ 정답: ③
허니토큰(Honeytoken)은 시스템이 아닌 데이터 단위의 함정이다. 가짜 AWS API 키·비밀번호·신용카드 번호·이메일 주소를 배치하고, 이 데이터가 사용되거나 접근되면 침해가 발생했음을 탐지한다. ① 허니넷, ② 저 인터랙션 허니팟, ④ 허니월(Honeywall).
PART 02

침입 방지 시스템 (IPS)

🛡️
SECTION 2-1

IPS란?

탐지를 넘어 실시간 차단까지 — 능동적 방어 시스템

IPS(Intrusion Prevention System, 침입 방지 시스템)는 IDS(탐지)의 기능에 실시간 차단·대응 기능을 추가한 보안 시스템이다. 네트워크 트래픽 경로에 인라인(In-line)으로 배치되어 공격 트래픽을 실시간으로 탐지하고 즉시 차단한다.

💡 IDS vs IPS 핵심 차이
IDS(탐지 시스템): 트래픽을 복사하여 분석 → 경보만 발생 (Out-of-band, 수동적)
IPS(방지 시스템): 트래픽 경로에 직렬 배치 → 탐지 즉시 차단 (In-line, 능동적)
IPS = IDS + 차단 기능

SECTION 2-4

IPS/IDS 탐지 기법

오용 탐지 vs 이상 탐지 — 두 가지 방식의 특성과 한계
📖
오용 탐지
Misuse Detection / Signature-Based

알려진 공격 패턴(시그니처)과 수신 트래픽을 비교하여 일치하면 탐지한다. 바이러스 백신의 시그니처 방식과 동일한 원리다.

오탐(FP) 비율 낮음
제로데이 탐지 불가
처리 속도 빠름
관리 부담 시그니처 업데이트
핵심: 알려진 공격만 탐지. 신규·제로데이 탐지 불가. 시그니처 지속 업데이트 필수.
📊
이상 탐지
Anomaly Detection / Behavior-Based

정상 행동의 기준선(Baseline)을 학습한 후, 이에서 크게 벗어나는 패턴을 이상으로 탐지한다. AI·통계 기반으로 동작한다.

오탐(FP) 비율 높음
제로데이 탐지 가능
처리 속도 보통~느림
학습 기간 필요
핵심: 미지 공격·제로데이 탐지 가능. 오탐 많아 IPS에 단독 사용 위험. 튜닝 중요.
📉

💡 IDS vs IPS에서 FP·FN의 영향 차이
IDS에서 오탐(FP): 불필요한 경보 → 관리자 피로감·업무 방해 (비교적 허용 가능)
IPS에서 오탐(FP): 정상 서비스가 차단됨 → 서비스 장애 발생 → 위험
미탐(FN): 공격이 탐지 안 되고 통과 → IDS·IPS 모두 위험

Q3 · 군무원 / 9급 공무원 유형
IDS와 IPS의 차이에 대한 설명으로 옳은 것은?
①IDS는 트래픽 경로에 In-line으로 배치되어 악성 트래픽을 즉시 차단한다.
②IPS는 트래픽의 복사본을 분석하여 공격을 탐지하고 경보를 발생시킨다.
③IDS는 Out-of-Band 방식으로 탐지 및 경보만 하고, IPS는 In-line으로 탐지 후 즉시 차단한다.
④IDS와 IPS는 배치 방식은 동일하며 차단 기능의 유무만 다르다.
✅ 정답: ③
IDS: Out-of-Band(트래픽 복사본 분석), 탐지+경보만, 직접 차단 불가
IPS: In-line(트래픽 경로 직렬 배치), 탐지+즉시 차단 가능
배치 방식이 근본적으로 다르다. ①②는 서로 IDS와 IPS의 설명이 반대로 되어 있다.
Q4 · 정보보호 기사 유형
IPS/IDS 탐지 기법 중 오용 탐지(Misuse Detection)에 대한 설명으로 옳은 것은?
①정상적인 행동의 기준선(Baseline)을 학습하고, 이에서 벗어나는 행동을 탐지한다.
②알려지지 않은 새로운 공격(제로데이)을 탐지하는 데 효과적이다.
③알려진 공격 패턴(시그니처)과 비교하여 일치하면 탐지하므로 오탐(FP)이 낮다.
④오탐(FP) 비율이 높아 IPS에 단독으로 적용하면 정상 서비스가 차단될 위험이 크다.
✅ 정답: ③
오용 탐지(Misuse Detection / Signature-Based): 알려진 공격 패턴(시그니처)과 일치할 때만 탐지하므로 오탐(FP)이 낮고 처리 속도가 빠르다. 단, 알려지지 않은 공격(제로데이)은 탐지 불가. ①④는 이상 탐지(Anomaly Detection)의 특징이다.
Q5 · 군무원 종합 유형
IPS에서 오탐(False Positive, FP)이 발생했을 때 나타나는 문제로 가장 적절한 것은?
①실제 공격이 탐지되지 않고 통과된다.
②정상적인 트래픽이 공격으로 오인되어 차단되므로 서비스 장애가 발생한다.
③새로운 공격 기법을 탐지하지 못하는 문제가 발생한다.
④허니팟으로 전달되어야 할 공격 트래픽이 실제 서버로 전달된다.
✅ 정답: ②
IPS는 In-line 배치로 실제 트래픽을 처리하므로 오탐(FP: 정상→공격 오판) 시 정상 트래픽이 차단된다. 이는 서비스 장애로 이어진다. 반면 IDS에서 오탐은 불필요한 경보만 발생하므로 서비스에 직접 영향을 주지 않는다. ①은 미탐(FN)의 문제다.
Q6 · 군무원 / 공무원 종합 유형
다음 보안 장치에 대한 설명으로 옳지 않은 것은?
①방화벽은 미리 정의된 정책에 따라 트래픽을 허용·차단하는 경계 보안 장치다.
②IPS는 In-line으로 배치되어 악성 트래픽을 실시간으로 탐지하고 차단할 수 있다.
③허니팟은 공격자를 유인하여 직접 차단하는 적극적 방어 시스템이다.
④IDS는 Out-of-Band 방식으로 탐지 및 경보만 발생시키며 직접 차단은 불가능하다.
✅ 정답: ③
허니팟은 공격자를 유인하고 관찰·기록하는 시스템이지, 직접 차단하지 않는다. 허니팟의 목적은 공격 기법 수집·위협 인텔리전스 생성이다. 직접 차단은 IPS와 방화벽의 기능이다. "적극적 차단"이 아닌 "능동적 정보 수집"이 허니팟의 특성이다.
