SECTION 01

VPN(가상 사설망)이란?

공개 인터넷 위에 암호화된 가상의 사설 터널을 구축하는 기술

VPN(Virtual Private Network, 가상 사설망)은 공개 네트워크(인터넷)를 통해 사설 네트워크처럼 안전하게 통신할 수 있도록 암호화된 터널을 생성하는 기술이다. 물리적으로 별도의 전용선 없이도 기밀성·무결성·인증을 보장하는 가상의 사설망을 구성한다.

SECTION 02

VPN 핵심 기술 요소

터널링 · 암호화 · 인증 · 키 교환
기술 요소역할대표 알고리즘·프로토콜
터널링 (Tunneling) 패킷을 다른 패킷으로 캡슐화하여 가상 터널을 통해 전송 IPSec, L2TP, PPTP, GRE, WireGuard
암호화 (Encryption) 데이터 내용을 암호화하여 기밀성 보장 AES-256(현대 표준), 3DES(구형), ChaCha20
인증 (Authentication) VPN 터널 양단의 신원 검증, 데이터 출처 확인 X.509 인증서, PSK(사전 공유 키), RADIUS, EAP
무결성 (Integrity) 전송 중 데이터 변조 탐지 HMAC-SHA256, HMAC-MD5(구형)
키 교환 (Key Exchange) 암호화 키를 안전하게 협상·교환 DH(Diffie-Hellman), ECDH, IKE(v1/v2)
💡 DH(Diffie-Hellman) 키 교환 — 공개 채널에서 비밀 키 생성
VPN에서 암호화 키를 어떻게 안전하게 공유할까? Diffie-Hellman 알고리즘은 공개 채널(도청 가능)을 통해서도 제3자가 알 수 없는 공유 비밀 키를 생성한다. A와 B가 각자의 비밀값과 공개값을 교환하면 동일한 세션 키를 도출할 수 있다. IPSec IKE 핸드셰이크의 핵심이다.

SECTION 03

터널링 (Tunneling) 원리

원본 패킷을 새로운 패킷으로 캡슐화(Encapsulation)하는 핵심 메커니즘

캡슐화(Encapsulation)는 원본 패킷 전체를 페이로드로 감싸 새로운 IP 헤더를 붙이는 과정이다. 공개 인터넷에서는 새 IP 헤더의 VPN 게이트웨이 주소로 라우팅되며, 목적지 VPN 게이트웨이에서 역캡슐화(Decapsulation)된다.

⚠️ 일반 IP 패킷 (암호화 없음)
IP 헤더 (출발지/목적지 IP)
TCP/UDP 헤더 (포트 번호)
페이로드 데이터 (평문 노출!)
🔒 VPN 캡슐화 패킷 (IPSec 터널 모드)
새 IP 헤더 (VPN GW IP)
IPSec ESP 헤더
🔐 암호화 영역
원본 IP 헤더 (숨겨짐)
원본 TCP/UDP 헤더
원본 데이터 (암호화)
ESP 트레일러 / ICV (무결성)
터널 모드 vs 전송 모드
터널 모드 (Tunnel Mode)
원본 패킷 전체 캡슐화
원본 IP 헤더까지 암호화하여 새 IP 헤더를 붙인다. 출발지·목적지 IP가 은닉된다. VPN 게이트웨이 간 또는 원격 접속에 주로 사용.
사용: Site-to-Site VPN, 원격 접속 VPN
전송 모드 (Transport Mode)
페이로드만 캡슐화
원본 IP 헤더는 그대로 두고 페이로드(TCP/UDP + 데이터)만 암호화한다. 출발지·목적지 IP는 노출된다. 종단 간(End-to-End) 보안에 사용.
사용: 호스트 간 직접 통신, L2TP 내부
📝 기출 유형
확인 문제 — VPN 개념·터널링
Q1 · 군무원 / 9급 공무원 유형
VPN(가상 사설망)에 대한 설명으로 옳지 않은 것은?
①공개 인터넷을 통해 암호화된 터널을 구성하여 사설망처럼 통신할 수 있다.
②터널링, 암호화, 인증을 핵심 기술 요소로 사용한다.
③VPN은 전용 물리적 회선을 사용하므로 일반 인터넷 회선보다 비용이 높다.
④데이터의 기밀성, 무결성, 인증을 보장한다.
✅ 정답: ③
VPN은 물리적 전용 회선을 사용하지 않는다. 기존 공개 인터넷 회선 위에 소프트웨어적으로 암호화 터널을 구성하므로 전용 전용선 대비 비용이 훨씬 저렴하다. 이것이 VPN의 가장 큰 장점 중 하나다.
Q2 · 정보보호 기사 유형
IPSec의 터널 모드(Tunnel Mode)와 전송 모드(Transport Mode)의 차이로 옳은 것은?
①터널 모드는 페이로드만 암호화하고, 전송 모드는 IP 헤더까지 암호화한다.
②터널 모드는 원본 IP 헤더까지 암호화하여 새 IP 헤더를 붙이고, 전송 모드는 페이로드만 보호한다.
③터널 모드는 UDP를 사용하고, 전송 모드는 TCP를 사용한다.
④터널 모드와 전송 모드는 암호화 범위가 동일하고 헤더 구조만 다르다.
✅ 정답: ②
터널 모드: 원본 패킷(헤더+페이로드) 전체를 암호화 후 새 IP 헤더 추가 → IP 주소 은닉, Site-to-Site 사용
전송 모드: 원본 IP 헤더는 유지, 페이로드만 암호화 → IP 주소 노출, End-to-End 사용

SECTION 04

IPSec (IP Security)

네트워크 계층(L3)에서 동작하는 가장 표준적인 VPN 프로토콜 스위트

IPSec은 IP 패킷에 대한 인증·암호화·무결성을 제공하는 프로토콜 집합이다. L3(네트워크 계층)에서 동작하므로 상위 응용 프로그램과 독립적으로 모든 IP 트래픽을 보호한다.

AH vs ESP 상세 비교

인증 헤더(AH)와 보안 페이로드(ESP)의 보호 범위 차이
✅
AH (Authentication Header)
IP 프로토콜 번호 51
인증과 무결성만 제공하며 암호화는 하지 않는다. IP 헤더를 포함한 전체 패킷에 무결성 검증을 적용하여 패킷이 전송 중 변조되지 않았음을 보장한다.
패킷 구조 (전송 모드)
IP 헤더 (변경 불가 필드 포함)
AH 헤더 (인증 데이터 포함)
페이로드 (평문! 암호화 안 됨)
✅ AH 무결성 검증 범위
IP 헤더(변경 불가 필드) + AH 헤더 + 페이로드 전체
✓ 인증 ✓ 무결성 ✗ 암호화 없음 ✗ NAT 통과 어려움
🔐
ESP (Encapsulating Security Payload)
IP 프로토콜 번호 50
암호화 + 인증 + 무결성을 모두 제공한다. VPN에서 가장 많이 사용되는 방식으로, 페이로드(데이터)를 암호화하여 기밀성을 보장하고 ICV(무결성 검증값)로 변조를 탐지한다.

💡 AH + ESP 동시 사용
AH와 ESP를 함께 사용하면 IP 헤더까지 무결성 보장(AH) + 페이로드 암호화(ESP)를 동시에 달성할 수 있다. 단, 이는 복잡도를 높이며 NAT 통과가 어렵다. 현대 환경에서는 ESP만 사용하는 것이 일반적이다.
📝 기출 유형
확인 문제 — IPSec AH / ESP
Q3 · 군무원 / 정보보호 기사 유형 ★★★
IPSec에서 AH(Authentication Header)와 ESP(Encapsulating Security Payload)의 차이에 대한 설명으로 옳은 것은?
①AH는 암호화 기능을 제공하고, ESP는 인증 기능만 제공한다.
②AH와 ESP 모두 IP 헤더를 포함한 전체 패킷을 암호화한다.
③AH는 암호화 없이 인증과 무결성만 제공하고, ESP는 암호화·인증·무결성을 모두 제공한다.
④ESP는 IP 프로토콜 번호 51을, AH는 50을 사용한다.
✅ 정답: ③
AH(IP 번호 51): 인증 + 무결성 / 암호화 없음 / NAT 통과 어려움
ESP(IP 번호 50): 암호화 + 인증 + 무결성 모두 제공 / NAT-T로 NAT 통과
④는 번호가 반대: AH=51, ESP=50
Q4 · 군무원 유형
IPSec에서 SA(Security Association)에 대한 설명으로 옳은 것은?
①양방향 통신에 SA 하나만 필요하다.
②SA는 VPN의 물리적 연결을 의미한다.
③SA는 단방향이므로 양방향 VPN 통신에는 SA가 2개 필요하다.
④SA는 IKE Phase 2에서만 생성된다.
✅ 정답: ③
SA(Security Association)는 단방향(Unidirectional)이다. 양방향 통신을 위해서는 A→B SA와 B→A SA, 총 2개의 SA가 필요하다. SA에는 암호화 알고리즘·키·모드 등 보안 매개변수가 저장된다. IKE Phase 1에서는 IKE SA, Phase 2에서는 IPSec SA가 생성된다.

SSL/TLS VPN

웹 브라우저만으로 접속 가능한 응용 계층 VPN — 원격 접속에 최적

SSL VPN은 HTTPS(SSL/TLS) 기반으로 동작하는 VPN이다. 별도의 클라이언트 소프트웨어 없이 웹 브라우저만으로 접속할 수 있어 원격 접속에 매우 편리하다. TCP 443번 포트를 사용하므로 방화벽 통과가 용이하다.

💡 SSL VPN vs IPSec VPN 선택 기준
SSL VPN: 재택근무자, 외부 직원 → 브라우저만으로 사내 웹 시스템 접근. 설치 간편.

SECTION 07

VPN 프로토콜 상세 비교

Q5 · 군무원 / 정보보호 기사 유형
SSL VPN에 대한 설명으로 옳지 않은 것은?
①응용 계층(L7)에서 동작하며 HTTPS 기반으로 구현된다.
②TCP 443번 포트를 사용하므로 방화벽 통과가 용이하다.
③전용 클라이언트 소프트웨어가 반드시 필요하므로 설치 부담이 크다.
④원격 재택근무 환경에서 웹 브라우저만으로 사내 시스템에 접근할 수 있다.
✅ 정답: ③
SSL VPN의 가장 큰 장점 중 하나가 웹 브라우저만으로 접속 가능하다는 것이다. 별도의 VPN 클라이언트 소프트웨어 설치 없이 HTTPS를 통해 접속한다. 별도 클라이언트가 필요한 것은 IPSec VPN이나 OpenVPN이다.
Q6 · 군무원 기출 유형
본사와 지사 간 항상 연결된 VPN 터널을 구성하여 양쪽 직원이 서로의 내부 시스템에 접근할 수 있도록 하는 VPN 구성 방식은?
①Remote Access VPN
②Site-to-Site VPN
③End-to-End VPN
④Cloud VPN
✅ 정답: ②
Site-to-Site VPN(거점 간 VPN): 두 거점의 VPN 게이트웨이 간 항상 유지되는 터널. 사용자가 VPN을 의식하지 않고 다른 거점 시스템에 접근. Remote Access는 개별 사용자→사내망, End-to-End는 호스트 간 직접 연결이다.
Q7 · 군무원 유형
VPN 프로토콜 중 보안이 취약하여 현재 사용을 지양해야 하는 레거시 프로토콜은?
①PPTP (Point-to-Point Tunneling Protocol)
②IKEv2/IPSec
③WireGuard
④OpenVPN
✅ 정답: ①
PPTP는 1990년대 개발된 레거시 VPN 프로토콜로, MPPE 암호화의 취약점이 다수 발견되어 현재는 보안 전문가들이 사용을 강력히 지양한다. Microsoft도 PPTP 사용을 권고하지 않는다. 현대 권장 프로토콜: IKEv2/IPSec, WireGuard, OpenVPN.

Q8 · 군무원 종합 유형
다음 VPN 관련 설명 중 옳지 않은 것은?
①IPSec AH는 IP 프로토콜 번호 51을 사용하며 암호화 기능은 제공하지 않는다.
②IKE는 UDP 500번 포트를 사용하여 IPSec 연결의 키 교환과 SA 협상을 수행한다.
③IPSec 터널 모드에서는 페이로드만 암호화되고 원본 IP 헤더는 평문으로 전송된다.
④SSL VPN은 웹 브라우저를 통해 접속할 수 있어 별도 클라이언트 소프트웨어가 필요하지 않다.
✅ 정답: ③
터널 모드는 페이로드만이 아니라 원본 IP 헤더까지 암호화하고 새로운 IP 헤더를 붙인다. 이 덕분에 출발지·목적지 IP 주소도 은닉된다. 페이로드만 보호하는 것은 전송 모드(Transport Mode)의 특징이다.
Q9 · 정보보호 기사 종합 유형
VPN에서 사용되는 Diffie-Hellman(DH) 알고리즘의 목적으로 옳은 것은?
①VPN 터널을 통해 전송되는 데이터를 암호화하기 위한 알고리즘이다.
②VPN 사용자의 신원을 인증하기 위한 알고리즘이다.
③공개 채널을 통해 도청자 없이 양측이 안전하게 공유 비밀 키를 협상하는 알고리즘이다.
④데이터의 무결성을 검증하기 위한 해시 알고리즘이다.
✅ 정답: ③
Diffie-Hellman(DH)은 키 교환(Key Exchange) 알고리즘이다. 공개 채널(도청 가능)을 통해 양측이 각자의 비밀값을 공개하지 않고도 동일한 공유 세션 키를 생성한다. IPSec IKE 핸드셰이크에서 세션 암호화 키를 안전하게 협상하는 데 사용된다.
Q10 · 군무원 종합 유형
다음 중 VPN 프로토콜과 동작 계층의 연결이 옳지 않은 것은?
①IPSec — 네트워크 계층(L3)
②SSL VPN — 응용 계층(L7)
③PPTP — 응용 계층(L7)
④L2TP — 데이터 링크 계층(L2)
✅ 정답: ③
PPTP는 응용 계층(L7)이 아닌 데이터 링크 계층(L2)에서 동작하는 프로토콜이다. PPP 프레임을 GRE로 캡슐화하여 L2 터널을 구성한다. 응용 계층에서 동작하는 VPN은 SSL VPN이다.
