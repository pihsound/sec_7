1
인터넷이 기업에 들어오던 날

1990년대 초, 기업들이 하나씩 인터넷에 연결되기 시작했어요. 전 세계와 연결된다는 흥분이 있었지만, 곧 불안이 따라왔어요. "인터넷에 연결한다는 건, 전 세계 누구든 우리 컴퓨터에 접근할 수 있다는 뜻 아닌가요?"

내부 네트워크는 사내 직원들이 쓰는 공간이에요. 여기엔 급여 데이터, 고객 정보, 내부 문서가 있어요. 인터넷과 연결하되, 외부에서 함부로 들어오지 못하게 막아야 했어요. 그래서 등장한 게 방화벽(Firewall)이에요.

방화벽의 아이디어는 단순해요.
"내부와 외부 사이에 문지기를 세우자. 허가된 것만 통과시키고 나머지는 막는다."
건물의 방화벽(불이 번지는 걸 막는 벽)에서 이름을 따왔어요.
2
방화벽만으론 부족했다 — IDS의 등장

방화벽을 쳤는데도 침입이 발생했어요. 이유가 있었어요. 방화벽은 "허가된 포트로 들어오는 것"은 통과시키거든요. 공격자가 허가된 포트(예: 80번 HTTP)를 통해 악성 트래픽을 보내면 방화벽이 못 막아요.

그래서 나온 게 IDS(침입탐지시스템)예요. 방화벽과 달리 IDS는 내용을 분석해요. "이 패킷이 허가된 포트로 왔어도, 내용이 공격 패턴과 일치하면 알람을 울린다."

방화벽 vs IDS의 역할 차이를 비유하면:
방화벽 = 건물 입구 경비원 (출입증 있으면 통과)
IDS = CCTV (들어온 사람이 이상한 행동을 하면 알람)
IPS = 무장 경비원 (이상한 행동 즉시 제압·차단)
3
출장 직원의 고민 — VPN의 탄생

인터넷이 보급되면서 새로운 고민이 생겼어요. 재택근무자나 출장 중인 직원이 본사 내부 시스템에 접근해야 하는데, 그러려면 인터넷을 통해야 해요. 근데 인터넷은 공개된 네트워크라 데이터가 노출돼요.

전용선을 깔면 안전하지만 비용이 엄청 비싸요. 서울-부산-뉴욕-런던을 전용선으로 연결한다고 생각해 보세요.

"공인 인터넷을 쓰되, 우리끼리만 읽을 수 있게 암호화해서 터널을 만들자."
이게 VPN(Virtual Private Network — 가상 사설망)이에요.
물리적으로는 공인 인터넷을 쓰지만, 논리적으로는 전용선처럼 작동해요.
4
IP가 부족해 — NAT와 SNMP

IPv4 주소는 약 43억 개예요. 처음엔 충분할 것 같았지만 인터넷 폭발로 금방 부족해졌어요. 회사 내 컴퓨터 100대가 각각 공인 IP를 쓰면 100개의 공인 IP가 필요해요. 너무 비싸고 부족해요.

NAT(Network Address Translation)는 이렇게 해결해요. 회사 내부에선 사설 IP(192.168.x.x)를 쓰고, 인터넷에 나갈 때만 공인 IP로 변환해요. 공인 IP 1개로 내부 수백 대를 인터넷에 연결 가능.

그리고 네트워크가 커질수록 "어디서 장애가 났는지 어떻게 알아요?"라는 문제가 생겼어요. SNMP는 네트워크 장비들의 상태를 중앙에서 모니터링하는 프로토콜이에요. 의사(NMS)가 환자(장비)의 건강을 원격으로 체크하는 구조예요.

1
방화벽 정의 및 기능
★★★
방화벽 정의

방화벽(Firewall)은 내부 네트워크와 외부 인터넷 사이에서 미리 정해진 보안 정책에 따라 트래픽을 허가/차단하는 시스템. 접근통제·사용자인증·감사로그·프라이버시보호·서비스통제 기능 제공. 내부 스파이·바이러스·방화벽 우회 트래픽은 방어 불가.

2
방화벽 4가지 유형
★★★ 최빈출
방화벽 4가지 유형 — OSI 계층별 동작 위치
①
패킷 필터링 (Packet Filtering)
네트워크/전송 계층(3~4층)에서 IP 헤더·포트 번호·TCP 플래그를 검사하여 허용/차단. 스크리닝 라우터라고도 불림. 가장 빠르고 구현 간단. 단점: 응용 계층 내용 분석 불가, 로그 기능 부족, TCP 연결 상태 추적 불가.
출발지IP·목적지IP·포트·플래그 검사
②
서킷 레벨 게이트웨이 (Circuit Level Gateway)
세션 계층(5층)에서 TCP 연결 상태를 검사. 최초 연결 수립 시 검증 후 이후 패킷은 신뢰. 내부 IP 구조 숨기기 가능. SOCKS 프로토콜 기반. 패킷 내용(응용 데이터)은 분석 안 함.
TCP 연결 상태 검사 / SOCKS
③
스테이트풀 패킷 검사 (Stateful Inspection)
패킷 필터링 + 연결 상태 추적. 상태 테이블(State Table)에 TCP 연결 정보를 기록하고 추적. 패킷 단독이 아닌 연결 컨텍스트 전체를 분석. 세션 하이재킹 방어 가능. 현대 방화벽의 기본 방식.
패킷 + 연결 상태 추적 / 세션하이재킹 방어
④
응용 계층 게이트웨이 (Application Level Gateway) ★ 가장 강력
OSI 응용 계층(7층)에서 동작. 프록시(Proxy) 서버 방식. HTTP·FTP·SMTP 등 응용 프로토콜 내용 전체를 분석. 가장 높은 보안 수준. 단점: 가장 느림, 각 서비스별 프록시 필요.
★ OSI 7층(응용층) 동작 / 프록시 / 가장 강력
⚠함정 — "패킷 필터링 방화벽이 OSI 7계층에서 동작한다" / "응용층에서 동작하는 것 = 스테이트풀"
OSI 7계층(응용층)에서 동작하는 방화벽 = 응용 계층 게이트웨이(ALG)
패킷 필터링이 검사할 수 없는 것 → TCP 연결 상태 (→ 스테이트풀이 담당)

패킷 필터링: 출발지/목적지 IP주소, 포트 번호, TCP 플래그 비트 검사 가능. TCP 연결 상태는 검사 불가.

IDS 정의 및 탐지 방법 2가지
★★★
IDS 정의

IPS(Intrusion Prevention System, 침입방지시스템)는 IDS처럼 내용을 분석하되, 탐지하면 즉시 차단까지 수행. 트래픽 경로에 인라인(Inline)으로 설치. 안티바이러스(시그니처) + 방화벽(차단)이 결합된 형태.

VPN 정의 및 특징
★★★
VPN 정의

VPN(Virtual Private Network, 가상 사설망)은 공중망(인터넷)을 이용하여 사설망과 같은 보안 효과를 얻기 위한 기술. 터널링+암호화로 구현. 전용선 대비 저렴. 기밀성·무결성·사용자 인증 제공.

3
NAT — 사설 IP ↔ 공인 IP 변환
★★★
NAT 정의

NAT(Network Address Translation)는 사설 IP 주소를 공인 IP 주소로 변환하는 기술. 공인 IP 부족 문제 해결이 주목적. 여러 개의 사설 IP → 하나 이상의 공인 IP로 변환. 내부 네트워크 구조를 외부에 노출하지 않는 보안 이점.

NAT 동작 원리 — 사설 IP → 공인 IP 변환
⚠함정 — "NAT는 공인 IP와 사설 IP가 1:1 매핑만 가능하다" (2016 국가직)
② "공인 IP와 NAT IP의 매핑이 1:1만 가능하다" → ✗ 오답

NAT의 핵심은 여러 사설 IP → 하나의 공인 IP로 변환(N:1). 변환 테이블에 IP 주소 + 포트 번호를 함께 저장하여 여러 내부 호스트가 하나의 공인 IP를 공유. 단순 1:1 매핑은 Static NAT만 해당.

1
SNMP 정의 및 구성
★★★
SNMP 정의

SNMP(Simple Network Management Protocol)는 TCP/IP 네트워크에서 장치를 관리하기 위한 프로토콜. UDP 161번(요청·응답), UDP 162번(Trap) 사용. NMS-Agent-MIB 구조.

SNMP 구조 — NMS(관리자) · Agent · MIB
구성요소역할비유

GetRequest
NMS가 Agent에게 정보 요청. UDP 161번.
GetResponse
Agent가 NMS 요청에 응답. UDP 161번.
SetRequest
NMS가 Agent 변수 값 설정.
Trap ★
Agent가 비정상 상황 발생 시 능동적으로 NMS에 알림. UDP 162번.
⚠함정 — "MIB는 객체 이름·유형·인코딩 일반 규칙을 정의한다" vs "SMI가 규칙 정의"
"MIB는 객체의 이름을 붙이고 객체의 유형을 정의하며, 객체와 값을 부호화하는 등의 일반적인 규칙을 정의한다" → ✗ 오답 (2020 국가직 기출)

일반적인 규칙(이름·유형·인코딩) 정의 = SMI(Structure of Management Information)의 역할. MIB = 관리 객체들의 데이터베이스(정보 저장소). 자주 출제되는 MIB vs SMI 혼동 함정.

① TCP 플래그 비트
② 송수신지 IP주소
③ 송수신지 포트 번호
④ TCP 연결 상태
정답 ④ — 패킷 필터링: IP주소·포트번호·TCP 플래그는 검사 가능. TCP 연결 상태(세션 추적) = 스테이트풀 방화벽의 역할. 패킷 필터링은 개별 패킷만 보고 연결 상태를 기억하지 않음.
📄 2016 국가직 9급 · NAT 1:1 매핑 오답★★★
NAT(Network Address Translation)에 대한 설명으로 옳지 않은 것은?
① 한정된 공인 IP주소 부족 문제의 해결이 가능하다.
② 공인 IP와 NAT IP의 매핑이 1:1만 가능하다.
③ 주소 변환 기능을 제공한다.
④ 내부 시스템에 네트워크 구조를 노출하지 않는 보안상의 이점을 제공한다.
정답 ② — NAT는 여러 사설 IP → 하나의 공인 IP 변환(N:1 PAT/NAPT)이 기본. IP+포트 번호 조합으로 다수 사설 IP를 하나의 공인 IP로 매핑 가능.

③ ㉠오용탐지, ㉡이상탐지, ㉢네트워크기반
④ ㉠오용탐지, ㉡이상탐지, ㉢호스트기반
정답 ③ — 알려진 패턴=오용탐지. 정상 프로파일 비교=이상탐지. 모든 패킷 캡처=네트워크 기반 IDS.

② WAN

④ VPN

📄 2019 국가직 9급 · VPN 터널링 프로토콜 (PPTP·L2F·L2TP)★★★
VPN 기법에서 MS에서 개발한 (㉡) 기법과 시스코에서 개발한 (㉢) 기법이 있으며, 이 둘을 결합한 형태로 (㉣) 기법이 있다.
① VPN, PPTP, L2TP, L2F
② VPN, PPTP, L2F, L2TP
③ PPTP, VPN, L2TP, L2F

정답 ② — ㉡MS=PPTP, ㉢시스코=L2F, ㉣PPTP+L2F=L2TP. VPN은 전체 기술명. L2TP = PPTP(MS) + L2F(Cisco) 결합.
📄 2020 국가직 9급 · SNMP MIB vs SMI★★★
SNMP(Simple Network Management Protocol)에 대한 설명으로 옳지 않은 것은?
① 관리자는 GetRequest와 같은 메시지를 에이전트에 보내서 에이전트의 정보를 요구한다.
② 에이전트는 비정상적인 상황을 관리자에게 경고하기 위하여 Trap 메시지를 관리자에 보낼 수 있다.
③ UDP 포트 161번과 162번을 사용한다.
④ MIB는 객체의 이름을 붙이고 객체의 유형을 정의하며, 객체와 값을 부호화하는 등의 일반적인 규칙을 정의한다.
정답 ④ — 객체 이름·유형·인코딩 일반 규칙 정의 = SMI. MIB = 관리 대상 객체들의 데이터베이스(저장소). SMI는 MIB의 정보를 기술하는 언어·규칙. 혼동 주의!
📄 2020 해경 9급 · IDS 이상탐지 vs 오용탐지 매핑★★★
○ (㉠) 기법은 정상적인 활동 데이터베이스와 비교하여 침입을 구분한다.
○ (㉡) 기법은 공격패턴·시그니처 데이터베이스와 비교하여 침입을 구분한다.
○ (㉢) IDS는 네트워크 상의 모든 패킷을 캡처한 후 이를 분석하여 침입을 탐지하는 시스템이다.
① ㉠이상탐지, ㉡오용탐지, ㉢네트워크기반
② ㉠이상탐지, ㉡오용탐지, ㉢호스트기반
③ ㉠오용탐지, ㉡이상탐지, ㉢네트워크기반
④ ㉠오용탐지, ㉡이상탐지, ㉢호스트기반
정답 ① — 정상 활동 DB 비교=이상탐지. 공격패턴 DB 비교=오용탐지. 패킷 캡처=네트워크기반. ⚠ 2016과 동일 패턴이지만 ㉠㉡이 반대로 서술됨. 정확히 읽어야 함.
📄 2021 국가직 9급 · 호스트기반 IDS 오답★★★
IDS(Intrusion Detection System)에 대한 설명으로 옳지 않은 것은?
① IDS는 네트워크기반 및 호스트기반으로 분류할 수 있다.
② 오용탐지기법은 시그니처기반 탐지 또는 지식기반 탐지로 불린다.
③ 네트워크기반 IDS는 네트워크 전반에 대한 감시를 할 수 있다.
④ 호스트기반 IDS는 시스템 로그에 대한 탐지보다는 네트워크 패킷을 수집하여 공격을 판단한다.
정답 ④ — 호스트기반 IDS = 시스템 로그·파일 변경·프로세스 분석이 주역할. 네트워크 패킷 수집 = 네트워크기반 IDS(NIDS).
📄 2021 해경 9급 · IPS 설명 오답★★
침입 방지 시스템에 대한 설명으로 가장 옳지 않은 것은?
① 침입 방지 시스템의 검사영역은 네트워크 계층과 전송계층의 IP/Port 정보를 기반으로 동작한다.
② 방화벽이 검사할 수 없는 전송계층 상단의 응용 계층 데이터까지도 검사가 가능하다.
③ 침입 경고 이전에 상대의 공격을 중단시키는데 목적을 두고 있다.
④ 안티바이러스 같은 시그니처 기반과 방화벽 같은 네트워크 차단 기능이 결합된 방식이다.
정답 ① — IPS는 응용 계층까지 검사 가능. IP/Port 기반만 동작하는 건 패킷 필터링 방화벽의 설명. IPS의 핵심은 방화벽보다 깊은 계층 분석 + 차단.
📄 2021 해경 9급 · VPN 4계층 프로토콜 오답★★★
VPN에 대한 설명으로 가장 옳지 않은 것은?
① 전용선보다 저렴하고 기밀성·무결성·사용자인증 보안 기능을 제공한다.
② 공공 네트워크를 통해 기업의 재택근무자·이동 직원이 안전하게 접근 가능하다.
③ 가상 터널이 형성되도록 하는 터널링 기능은 SSH와 같은 OSI 모델 4계층의 보안 프로토콜로 구현해야 한다.
④ 공중망을 이용하여 사설망과 같은 효과를 얻기 위한 기술이다.
정답 ③ — VPN 터널링은 2계층(PPTP·L2TP)이나 3계층(IPSec)에서 구현. "4계층 SSH"로만 구현해야 한다는 건 틀림. SSH는 응용 계층(7층) 프로토콜이기도 함.
📄 2023 해경 9급 · DMZ 설명★★
DMZ(demilitarized zone)에 대한 설명으로 가장 옳은 것은?
① 외부 네트워크에서는 DMZ에 접근할 수 없다.
② DMZ 내에는 주로 내부 접속용 데이터베이스 서버가 위치한다.
③ 내부 사용자가 DMZ에 접속하기 위해서는 외부 방화벽을 거쳐야 한다.
④ DMZ는 내부 방화벽과 외부 방화벽 사이에 위치할 수 있다.
정답 ④ — DMZ = 외부FW + 내부FW 사이. ①DMZ는 외부에서 접근 가능(웹서버 있으니까). ②DB서버는 내부, DMZ엔 웹·메일·DNS 서버. ③내부→DMZ는 내부 방화벽을 거침.
