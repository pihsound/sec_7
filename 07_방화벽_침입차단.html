회선 레벨 대표: SOCKS 프록시
📝 기출 유형
확인 문제 — 방화벽 분류
Q1 · 군무원 / 9급 공무원 유형
침입 차단 시스템(방화벽)의 차단 방법에 따른 분류 중, 가장 빠른 처리 속도를 가지지만 IP 스푸핑에 취약하고 연결 상태를 추적하지 않는 방식은?
①패킷 필터링 게이트웨이
②회선 레벨 게이트웨이
③응용 레벨 게이트웨이
④혼합형 게이트웨이
✅ 정답: ①
패킷 필터링 게이트웨이는 L3·L4 헤더(IP·포트·프로토콜)만 검사하는 Stateless 방식이다. 처리 속도가 가장 빠르지만, 연결 상태를 추적하지 않아 IP 스푸핑·단편화 공격에 취약하고 패킷 내용은 검사하지 못한다.
Q2 · 정보보호 기사 / 군무원 유형
방화벽의 구성 방법 중 이중 방화벽과 DMZ(비무장지대)를 구성하여 보안성이 가장 높은 방식은?
①배스천 호스트 (Bastion Host)
②스크린드 서브넷 (Screened Subnet)
③스크린드 호스트 게이트웨이 (Screened Host Gateway)
④이중 홈 게이트웨이 (Dual-Homed Gateway)
✅ 정답: ②
스크린드 서브넷은 외부 방화벽→DMZ→내부 방화벽의 이중 방화벽 구조로 가장 높은 보안을 제공한다. DMZ에 공개 서버를 배치하여 서버 침해 시 내부망 확산을 방지한다. 단일 실패점(SPOF)이 없는 유일한 구성이다.
Q3 · 군무원 기출 유형
이중 홈 게이트웨이(Dual-Homed Gateway)에 대한 설명으로 옳은 것은?
①두 대의 독립된 방화벽 사이에 DMZ를 구성하는 방식이다.
②스크리닝 라우터와 배스천 호스트를 조합한 구성이다.
③두 개의 네트워크 인터페이스를 가진 배스천 호스트에서 IP 포워딩을 비활성화하여 내외부 망을 분리한다.
④단일 라우터에 패킷 필터링 기능을 추가한 가장 단순한 구성이다.
✅ 정답: ③
이중 홈 게이트웨이의 핵심: NIC 2개(외부·내부 각각) + IP 포워딩(IP Forwarding) 비활성화. IP 포워딩을 끄면 패킷이 두 NIC 사이를 직접 통과할 수 없어 모든 통신이 반드시 배스천 호스트의 응용 프로세스를 통해야 한다.
Q4 · 군무원 / 공무원 유형
방화벽 차단 방법 중 회선 레벨 게이트웨이(Circuit Level Gateway)의 특징으로 옳은 것은?
①IP 주소와 포트 번호만 검사하는 Stateless 방식이다.
②응용 계층까지 패킷 내용을 완전히 분석하는 가장 안전한 방식이다.
③TCP 3-way handshake의 유효성을 검증하며, SOCKS 프록시가 대표적 구현이다.
④패킷 필터링과 응용 프록시를 결합한 현대 방화벽의 주류 방식이다.
✅ 정답: ③
회선 레벨 게이트웨이는 세션 계층(L5)에서 TCP 3-way handshake의 유효성을 검증한다. TCP 연결이 성립된 후에는 패킷 내용을 검사하지 않고 릴레이한다. 대표 구현이 SOCKS 프록시(v5)이다.
Q5 · 종합 유형
다음 중 방화벽 분류에 대한 설명이 옳지 않은 것은?
①응용 레벨 게이트웨이는 패킷 내용을 검사할 수 있어 보안이 가장 높지만 처리 속도가 느리다.
②스크린드 서브넷은 단일 실패점(SPOF)이 없고 보안성이 가장 높은 구성이다.
③배스천 호스트 구성은 이중 방화벽을 사용하므로 단일 실패점이 발생하지 않는다.
④패킷 필터링 게이트웨이는 처리 속도가 가장 빠르지만 IP 스푸핑에 취약하다.
✅ 정답: ③
배스천 호스트는 단일 방화벽 시스템 하나로 구성되므로 단일 실패점(SPOF)이 존재한다. 이중 방화벽을 사용하는 구성은 스크린드 서브넷(Screened Subnet)이다. 배스천 호스트는 가장 단순하고 저렴하지만 SPOF가 단점이다.
