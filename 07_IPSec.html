1
IPSec 정의 및 특징
Definition
공식 정의

IPSec(IP Security)은 현재 사용 중인 인터넷 프로토콜(IPv4·IPv6)에 추가적인 보안 헤더를 붙여서 보안 기능을 제공하는 3계층(네트워크층) 표준 프로토콜이다. 인증·기밀성·키 관리 등 3가지 영역에서 사용할 수 있으며, 방화벽·게이트웨이·라우터에 구현한다. IP 스푸핑·스니핑에 대한 대응이 가능하다.

동작 계층
네트워크층(3층, IP 계층)에서 동작. 상위 응용 계층에 투명하게 보안 제공.
3대 기능 영역
① 인증(Authentication)
② 기밀성(Confidentiality)
③ 키 관리(Key Management)
구현 위치
방화벽, 라우터, 보안 게이트웨이, 호스트 등 다양한 위치에 구현 가능.
알고리즘
다양한 암호화 알고리즘 지원. 신규 알고리즘 적용 가능(지정된 알고리즘만 가능 → ✗ 오답)
적용 범위
원격 접속, 인트라넷, 엑스트라넷 방식 모두 적용 가능. 전자상거래 보안 강화에도 활용.
응용 투명성
응용 프로그램이 IPSec 존재를 모르고 동작. OS·응용 수정 불필요 (↔ SSL은 응용층에서 처리)

AH (Authentication Header) 완전 정리
★★★
AH 정의

AH(Authentication Header)는 IPSec의 인증 헤더 프로토콜로, IP 패킷에 대한 무결성·인증·재전송 방지를 제공한다. 단, 기밀성(암호화)은 제공하지 않는다. 인증 범위는 IP 헤더 포함 전체(터널 모드) 또는 IP 헤더 제외 나머지(전송 모드)에 적용 가능하다.

AH 패킷 구조 (IPv4 전송 모드)
AH 헤더 필드 구조
AH가 제공하는 서비스 ✓
무결성메시지 위변조 방지 (비연결 무결성)
인증데이터 발신처 인증
재전송재전송 공격 방지 (순서번호 사용)
접근제어SA 기반 접근 제어
AH가 제공하지 않는 서비스 ✗
기밀성암호화 없음! (ESP만 제공)
주의"AH 프로토콜의 페이로드 데이터와 패딩 내용은 기밀성 범위에 속한다" → ✗ 오답
혼동"AH는 암호화, ESP는 인증" → ✗ 반대!

1
ESP (Encapsulating Security Payload)
★★★
ESP 정의

ESP(Encapsulating Security Payload)는 암호화(기밀성) + 인증 + 무결성 + 재전송 방지를 모두 제공하는 IPSec 프로토콜이다. DES, 3-DES, IDEA, RC5 등의 암호화 알고리즘을 사용하며, 전송 모드와 터널 모드 두 가지 모두 적용 가능하다. AH와의 핵심 차이는 기밀성(암호화) 제공 여부이다.

⚠ESP 함정 — "ESP가 원래 IP패킷 전부를 암호화" (2023 지방직)

2023 지방직: "IPSec 터널 모드를 이용한 VPN에 대한 설명으로 옳지 않은 것은?"

③ "ESP는 원래 IP 패킷 전부와 원래 IP 패킷 앞뒤로 붙는 ESP 헤더와 트레일러를 모두 암호화한다" → ✗ 오답

터널 모드 ESP 암호화 범위: 원래 IP 패킷(헤더+페이로드) + ESP 트레일러까지 암호화. ESP 헤더 자체는 암호화 안 됨 (SPI·Seq# 인식 필요). ESP Auth(인증 데이터)는 암호화 범위 바깥(패킷 끝에 추가).

SA · IKE · SAD
Management
SA (Security Association)
AH·ESP 사용을 위해 통신 양측이 협의하는 단방향 논리적 관계. 양방향 통신에는 SA가 2개 필요. (SPI, 목적지 주소, 프로토콜)로 식별.
IKE (Internet Key Exchange)
IPSec SA를 자동으로 생성·협상하기 위한 프로토콜. ISAKMP 기반. SA 생성 = IKE 사용. OSPF가 아님!
SAD (Security Association Database)
SA를 모아놓은 데이터베이스. 발신·수신 패킷 처리 시 해당 SA를 찾기 위해 참조.
ISAKMP
Internet Security Association and Key Management Protocol. 인터넷에 대한 SA 관리 구조를 지원하고 프로토콜 제공. IKE가 SA 생성을 위해 사용.

⚠IKE 함정 — SA 생성에 "OSPF" 사용? (2020 국가직 7급)

2020 국가직 7급: "IPSec의 (㉠)는 발신지 인증과 데이터 무결성·기밀성을 제공. SA 생성을 위해 (㉡) 프로토콜을 사용"

㉠=ESP, ㉡=IKE → 정답 ③

오답 선지 ①·②: ㉡에 OSPF 삽입 → ✗. OSPF는 라우팅 프로토콜. SA 생성 = IKE

2018 국가직 IPSec 오답: "IKE는 IPSec의 보안 연관(SA)을 수립하고 필요한 보안키 교환 기능을 제공하지만, 인증서를 통한 개체 인증은 제공하지 않는다" → ✗ 오답. IKE는 인증서를 통한 인증도 지원함.

② 터널링을 통해 패킷들을 안전하게 전달하는 기능을 제공한다.

📄 2016 국가직 9급 · ESP에서만 제공하는 서비스★★★
IPSec 프로토콜의 AH와 ESP에서 제공하는 보안서비스 중 ESP에서만 제공하는 보안서비스는?
① 재전송 공격 방지 (Replay Attack Protection)
② 메시지 기밀성 (Confidentiality)
③ 메시지 무결성 (Integrity)
④ 개체 인증 (Entity Authentication)
정답 ② — 기밀성(암호화)=ESP만. 재전송방지·무결성·인증은 AH와 ESP 모두 제공. ④ 개체 인증도 AH·ESP 모두 가능. 이 문제는 IPSec 최빈출 유형으로 암기 필수.

IPsec에 대한 설명으로 옳지 않은 것은?
① IPsec 정책 설정 과정에서 송·수신자의 IP주소를 입력한다.
② AH(Authentication Header) 프로토콜은 무결성을 제공한다.
③ 트랜스포트(Transport) 모드에서는 IP헤더도 암호화된다.
④ 재전송 공격을 막기 위해 IP패킷별로 순서번호를 부여한다.
⑤ IKE(Internet Key Exchange) 프로토콜로 세션키를 교환한다.
정답 ③ — 전송 모드에서 암호화되는 것은 IP 페이로드(상위 계층 데이터). IP 헤더는 암호화 안 됨. IP 헤더까지 암호화(보호)하는 것은 터널 모드.
📄 2017 지방직 9급 · AH 기밀성 오답★★
IPSec 표준은 패킷을 보호하기 위해 AH와 ESP로 구성된다. 옳지 않은 것은?
① AH 프로토콜의 페이로드 데이터와 패딩 내용은 기밀성 범위에 속한다.
② AH 프로토콜은 메시지의 무결성을 검사하고 재연(Replay) 공격 방지 서비스를 제공한다.
③ ESP 프로토콜은 메시지 인증 및 암호화를 제공한다.
④ ESP는 전송 및 터널 모드를 지원한다.
정답 ① — AH는 기밀성(암호화)을 제공하지 않는다. AH의 페이로드 데이터와 패딩은 인증 범위에 속하지 기밀성 범위에 속하지 않음. 기밀성 = ESP만의 기능.
📄 2020 국가직 7급 · ESP + IKE 빈칸★★★
괄호 안에 들어갈 용어를 바르게 연결한 것은?

IPSec의 ( ㉠ )는 발신지 인증과 데이터 무결성 그리고 데이터 기밀성을 제공한다. 두 호스트 사이의 논리적 관계인 SA(Security Association)를 생성하기 위하여 ( ㉡ ) 프로토콜을 사용하여 보안상 안전한 채널을 확보한다.
① AH / OSPF
② AH / IKE
③ ESP / IKE
④ ESP / OSPF
정답 ③ — ㉠: 기밀성(암호화) 포함 = ESP. ㉡: SA 생성 = IKE. OSPF는 라우팅 프로토콜이므로 오답. AH는 기밀성 없음.
📄 2022 군무원 9급 · 전송 모드 옳은 설명★★★
IPSec의 전송모드에 대한 설명으로 가장 옳은 것은?
① IP패킷 전체를 보호한다.
② IP헤더를 보호하지 않으며 전송층에서 발송한 정보만 보호한다.
③ IP패킷 전체를 보호한 후에 새로운 IP헤더를 추가한다.
④ 전송층에서 온 정보에 IP헤더만 추가하고 IPSec헤더를 나중에 추가한다.
정답 ② — 전송 모드: IP 헤더 보호 안 함, 전송층(TCP/UDP) 이상 정보만 보호. ①③은 터널 모드 설명. ④는 잘못된 서술.
📄 2023 지방직 9급 · 터널 모드 VPN + ESP 암호화 범위★★
IPSec의 터널 모드를 이용한 VPN에 대한 설명으로 옳지 않은 것은?
① 양측 호스트 IP 주소를 숨기고 새로운 IP 헤더에 VPN 게이트웨이 IP를 넣는다.
② 터널 모드는 새로운 IP 헤더를 추가하기 때문에 전송 모드 대비 전체 패킷이 길어진다.
③ ESP는 원래 IP 패킷 전부와 원래 IP 패킷 앞뒤로 붙는 ESP 헤더와 트레일러를 모두 암호화한다.
④ ESP 인증 데이터는 패킷의 끝에 추가되며, 터널 모드의 경우 인증은 목적지 게이트웨이에서 이루어진다.
정답 ③ — 터널 모드 ESP 암호화 범위: 원래 IP패킷(헤더+페이로드) + ESP 트레일러. ESP 헤더 자체는 암호화 안 됨(SPI·Seq# 읽을 수 있어야 함). ESP Auth도 암호화 범위 밖.
