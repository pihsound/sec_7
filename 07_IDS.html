1
IDS 정의 및 방화벽과의 차이점
Definition
공식 정의

침입탐지시스템(IDS: Intrusion Detection System)은 네트워크 또는 시스템에 대한 불법 침입 행위를 실시간으로 감지·분석하여 관리자에게 경보를 발생시키는 시스템이다. 방화벽과 달리 패킷을 차단하지 않고 탐지·알림에 집중한다. 방화벽과 상호 보완적으로 운용한다.

IDS 탐지 가능한 공격
스캐닝취약점 찾기 위한 수동적 공격
DoS서비스 가용성 해치는 공격
침투버퍼 오버플로 등 시스템 권한 획득
내부자내부 사용자의 비정상 행위 탐지
IDS vs 방화벽 핵심 차이
역할방화벽=차단 / IDS=탐지(경보)
위치방화벽=경계 / IDS=사용자 네트워크 내부
정책방화벽=허용만 허용 / IDS=명백히 나쁜 것만 금지
차단방화벽=차단 / IDS=차단 불가 (탐지만)
⚠"IDS는 공격을 사전에 예방하고 차단할 수 있다" → ✗ 오답 (2018 경찰)
"IDS를 이용하면 공격 시도를 사전에 예방하고 차단할 수 있다" → ✗ 오답

IDS는 탐지(Detection)와 경보(Alert)가 목적이지 차단이 목적이 아니다. 차단 기능을 추가한 것이 IPS(침입방지시스템)이다. IDS 자체는 패킷을 차단하지 않는다.

"침입탐지시스템은 패킷을 차단하며, DMZ에서만 설치가 가능하다" → ✗ 오답 (2020 경찰)

IDS는 패킷을 차단하지 않으며, 방화벽 앞/뒤, 내부망, 백본 등 다양한 위치에 설치 가능하다.

탐지 기법 분류 — 오용탐지 vs 이상탐지
★★★ 최빈출
분류 기준 1탐지 기법(분석 방법)에 따른 분류
탐지 기법 ①
오용 탐지 (Misuse Detection)
= 시그니처 기반 / 지식 기반
정의이미 알려진 공격 패턴(시그니처)을 미리 DB에 저장해두고, 해당 패턴이 탐지되면 침입으로 판단
다른 이름시그니처 기반 탐지, 지식 기반 탐지, 패턴 매칭
오탐률(FP)낮음 — 알려진 패턴만 탐지하므로
장점구현 쉬움. 속도 빠름. False Positive 낮음
단점새로운(Zero-day) 공격 탐지 불가. DB 지속 업데이트 필요
기법 예시조건부 확률, 전문가 시스템, 상태 천이 분석, 패턴 매칭, 키스트로크 모니터링
탐지 기법 ②
이상 탐지 (Anomaly Detection)
= 비정상 탐지 / 통계 기반
정의정상적인 활동 프로파일을 생성하고, 이와 통계적으로 크게 벗어나는 행위를 침입으로 판단
다른 이름비정상 탐지, 통계적 탐지, 행위 기반 탐지
오탐률(FP)높음 — 정상 행위도 비정상으로 오판 가능
장점알려지지 않은 새로운 공격 탐지 가능 (Zero-day 대응)
단점False Positive(오탐) 높음. 임계치 설정 어려움
기법 예시통계적 분석, 특성 추출, 예측 가능한 패턴 생성, 신경망(Neural Network)
구분오용 탐지 (Misuse)이상 탐지 (Anomaly)
다른 이름시그니처 기반 / 지식 기반비정상 탐지 / 통계 기반
탐지 방법알려진 패턴과 매칭정상 프로파일과 비교
오탐률(FP)낮음높음
미탐률(FN)높음 (새 공격 탐지 못함)낮음
새 공격 탐지불가 ✗가능 ✓
속도빠름느림 (학습 필요)
임계치 설정쉬움어려움 ★ 시험 출제
DB 업데이트필요 (새 패턴 지속 추가)불필요 (자동 학습)
⚠오용탐지 함정 — "임계치 설정이 어렵다" (2018 해경)

2018 해경 기출: "IDS 공격탐지 방법 중 오용탐지에 대한 설명으로 가장 틀린 것은?"

④ "정상인지, 비정상인지 결정하는 임계치 설정이 어렵다" → ✗ 오답

임계치 설정이 어려운 것은 이상 탐지의 단점이다. 오용 탐지는 알려진 패턴과 매칭하는 방식이므로 임계치 개념이 없거나 설정이 상대적으로 쉽다.

⚠이상탐지 함정 — "낮은 오탐률" (2022 국가직 7급)
"비정상행위(Anomaly) 침입탐지는 오용(Misuse) 침입탐지 대비 낮은 오탐률을 보이지만 알려지지 않은 새로운 공격에 취약하다" → ✗ 오답

이상 탐지는 오용 탐지보다 오탐률(FP)이 높다. 낮은 오탐률은 오용 탐지의 특성이다. 또한 새로운 공격 탐지는 이상 탐지가 가능(장점)하고 오용 탐지가 취약(단점)하다. 이 선지는 두 가지가 모두 반대로 서술됨.

3
오탐(False Positive) / 미탐(False Negative)
★★★ 기출 단골
오탐 — False Positive (FP)

= 긍정 오류 = 과탐지

실제 = 정상 → 시스템 판단 = 공격

합법적 사용자를 침입자로 오판
정상 트래픽을 공격으로 탐지
"공격이 아닌데 공격이라고 판단"

이상 탐지에서 높게 나타남

미탐 — False Negative (FN)

= 부정 오류 = 미탐지

실제 = 공격 → 시스템 판단 = 정상

침입자를 합법적 사용자로 오판
실제 공격을 탐지하지 못함
"공격인데 공격이 아니라고 판단"

오용 탐지에서 새 공격 시 높게 나타남

구분False Positive (FP, 오탐)False Negative (FN, 미탐)
다른 이름긍정 오류, 과탐지부정 오류, 미탐지
실제 상황공격이 아닌 정상 상황실제 공격 상황
시스템 판단"공격"으로 오판"정상"으로 오판
결과합법 사용자를 침입자로 차단실제 침입을 탐지하지 못함
영향서비스 장애, 사용자 불편보안 사고 발생
어느 탐지에서 높음이상 탐지(Anomaly)오용 탐지(Misuse) — 새 공격 시
⚠FP/FN 혼동 함정 — 가장 자주 출제

2018 해경 기출: "IDS 탐지기능에서 False Negative의 의미로 가장 올바른 것은?"

② false(-)로 표현되고, 공격인데도 공격이 아니라고 오판 → ✓ 정답

헷갈리는 선지:

① false(+)로 표현되고, 공격이 아닌데도 공격이라고 오판 → ✗ 이것은 False Positive

2017 지방직: "오용탐지기법은 상대적으로 ( )가 높고... 이상탐지기법은 ( )가 높다"
→ 오용탐지 = False Negative 높음 / 이상탐지 = False Positive 높음

핵심 트레이드오프: 침입자 탐지 기준을 느슨하게 하면 → False Positive(오탐) 증가, False Negative(미탐) 감소. 기준을 엄격하게 하면 → False Negative 증가, 탐지 못하는 공격 증가. 두 오류는 반비례 관계.

★★★
분류 기준 2모니터링 대상(설치 위치)에 따른 분류
모니터링 분류 ①
호스트 기반 IDS (HIDS)
Host-based IDS
설치개별 호스트(서버/PC)에 각각 설치
탐지 대상시스템 로그, 감사 기록, 시스템 자원 사용

암호화암호화된 트래픽 탐지 어려움
단점트래픽 크게 증가 시 성능 저하. 침입 증거 수집 어려움
설치 위치방화벽 앞(외부) / 방화벽 뒤 / 내부 네트워크

⚠HIDS 함정 — "네트워크 패킷을 수집하여 공격 판단" (2021 국가직)

2021 국가직 기출: "IDS에 대한 설명으로 옳지 않은 것은?"

④ "호스트기반 침입탐지시스템은 시스템의 로그에 대한 탐지보다는 네트워크 패킷을 수집하여 공격을 판단한다" → ✗ 오답

HIDS는 시스템 로그, 감사 기록, 자원 사용 정보를 분석하는 것이지 네트워크 패킷 수집은 NIDS의 특성이다.

⚠NIDS 오탐지/이상탐지 설명 매칭 — 최빈출 빈칸 문제

2016 국가직, 2020 해경 동일 유형 출제:

( ) 기법은 이미 알려진 공격 패턴을 근거로 침입을 탐지
( ) 기법은 정상적인 활동 프로파일과 비교하여 벗어나는 행위를 탐지
( ) IDS는 네트워크 상의 모든 패킷을 캡처한 후 분석하는 시스템

정답: 오용탐지 / 이상탐지 / 네트워크 기반(NIDS)

→ 선지 ③이 항상 정답 (오용탐지, 이상탐지, 네트워크기반 순서)

6
IDS 구성 3단계
Process
1
정보 수집부 (Data Collection)
감사 데이터(Raw Data) 수집. 시스템 로그, 네트워크 패킷, 파일 시스템 상태 등 다양한 정보 수집.
2
정보 분석부 — 3단계
① 정보 가공·축약 (Data Reduction & Filtering): 수집된 정보를 패턴에 의해 분류·정제
② 분석·침입탐지 (Analysis & Intrusion Detection): 가공된 데이터로 침입 여부 판단
③ 보고·조치 (Reporting & Response): 침입 발생 시 관리자에게 경고
3
관리부 (Management)
분석 결과에 대한 통제·관리. 보안 정책 수립. IDS 시스템 전체 관리·운용.
IDS 주요 기능: 시스템 모니터링·분석 / 시스템 감사 기록 / 중요 데이터 무결성 평가 / 알려진 공격 패턴 인식 / 비정상 행위 통계 분석 / 운영체제 감사 추적 관리

IDS vs 방화벽 vs IPS 비교
3자 비교
구분방화벽 (Firewall)IDS (침입탐지)IPS (침입방지)
주요 역할차단탐지(경보)탐지 + 차단
동작 방식수동 (passive)수동 (passive)능동 (active)
패킷 차단가능불가가능
패킷 분석헤더(IP·포트) 기반헤더+내용(Payload)헤더+내용
탐지 영역네트워크·전송층(IP·포트)응용층까지 전체네트워크층+전송층
설계 정책허용할 것만 허용명백한 공격만 탐지알려진 공격 차단
네트워크 부하있음낮음있음
내부 공격방어 어려움탐지 가능(HIDS)부분 가능
⚠IPS 탐지 영역 함정 (2022 군무원 9급)

2022 군무원: "침입방지시스템(IPS)에 대한 설명으로 가장 옳지 않은 것은?"

② "탐지영역은 네트워크 계층과 전송계층이다" → ✗ 오답

IPS는 방화벽이 검사할 수 없는 전송계층 상단의 응용 계층 데이터까지도 검사 가능하다. 단순히 네트워크·전송층만이 아니다. (2021 해경 참조)

○ ( ㉠ ) 기법은 이미 알려진 공격 패턴을 근거로 침입을 탐지
○ ( ㉡ ) 기법은 정상적인 활동 프로파일과 비교하여 벗어나는 행위를 탐지
○ ( ㉢ ) IDS는 네트워크 상의 모든 패킷을 캡처링한 후 분석하여 탐지
① 이상탐지 / 오용탐지 / 네트워크 기반
② 이상탐지 / 오용탐지 / 호스트 기반
③ 오용탐지 / 이상탐지 / 네트워크 기반
④ 오용탐지 / 이상탐지 / 호스트 기반
정답 ③ — 알려진 패턴 매칭=오용탐지. 정상 프로파일 비교=이상탐지. 패킷 전체 캡처·분석=네트워크 기반(NIDS). 이 3개 매칭이 핵심이며 2020 해경에서도 동일 유형 출제됨.
📄 2017 지방직 9급 · 정보보호론 문19 — 오탐/미탐 빈칸★★★
침입 탐지 시스템의 탐지분석 기법에 대한 설명이다. ㉠~㉣에 들어갈 내용이 바르게 연결된 것은?

( ㉠ )은 알려진 공격 패턴을 미리 입력해두었다가 해당 패턴이 탐지되면 알려주는 것이다. 상대적으로 ( ㉡ )가 높고, 새로운 공격 탐지에 부적합하다. ( ㉢ )은 정상·평균 상태를 기준으로 급격한 변화나 확률 낮은 일이 발생하면 탐지로 알려준다. 상대적으로 ( ㉣ )가 높다.
① 이상탐지기법 / False Positive / 오용탐지기법 / False Negative
② 이상탐지기법 / False Negative / 오용탐지기법 / False Positive
③ 오용탐지기법 / False Negative / 이상탐지기법 / False Positive
④ 오용탐지기법 / False Positive / 이상탐지기법 / False Negative
정답 ③ — 오용탐지 = FN 높음(새 공격 탐지 못함). 이상탐지 = FP 높음(정상을 공격으로 오판). ㉠=오용탐지, ㉡=FN, ㉢=이상탐지, ㉣=FP.
📄 2018 해경 9급 — False Negative 의미★★★
IDS 탐지기능에서 False Negative의 의미로 가장 올바른 것은?
① false(+)로 표현되고, 공격이 아닌데도 공격이라고 오판
② false(-)로 표현되고, 공격인데도 공격이 아니라고 오판
③ false(+)로 표현되고, 공격인데도 공격이 아니라고 오판
④ false(-)로 표현되고, 공격이 아닌데도 공격이라고 오판
정답 ② — False Negative(미탐) = false(-) = 실제 공격인데 공격이 아니라고 오판. False Positive(오탐) = false(+) = 정상인데 공격이라고 오판. 2020 경찰에도 같은 내용 출제.
📄 2018 해경 9급 — 오용탐지 설명 오답★★
IDS 공격탐지 방법 중 오용탐지에 대한 설명으로 가장 틀린 것은?
① 시그니처 기반으로 동작한다.
② 속도가 빠르며, 구현이 쉽다.
③ 이미 정립된 공격패턴을 미리 입력하고 매칭하여 침입을 판단한다.
④ 정상인지, 비정상인지 결정하는 임계치 설정이 어렵다.
정답 ④ — 임계치 설정이 어려운 것은 이상 탐지(Anomaly Detection)의 단점. 오용 탐지는 알려진 패턴과 매칭하므로 임계치 개념이 다름. ①②③은 모두 오용탐지의 옳은 설명.
📄 2018 경찰공무원 · 시스템네트워크보안 문6 — IDS 차단 불가★★★
침입탐지시스템(IDS)에 대한 설명으로 가장 적절하지 않은 것은?
① 알려지지 않은 공격 행위도 이상탐지(Anomaly detection) 방식을 이용하여 탐지할 수 있다.
② IDS는 방화벽과 상호 보완적으로 사용될 수 있다.
③ 기존의 공격 패턴을 분석하여 침입을 탐지하는 오용탐지(Misuse detection) 방식을 사용할 수 있다.
④ IDS를 이용하면 공격 시도를 사전에 예방하고 차단할 수 있다.
정답 ④ — IDS는 탐지·경보가 목적. 차단·예방은 IPS(침입방지시스템)의 역할. IDS는 패킷 차단 기능 없음.
📄 2020 해경 9급 — 이상/오용/NIDS 매칭★★★
㉠~㉢에 들어갈 용어를 바르게 연결한 것은?
· ( ㉠ ) 기법은 정상적인 활동 데이터베이스와 비교하여 침입을 구분
· ( ㉡ ) 기법은 공격패턴·시그니처 데이터베이스와 비교하여 침입을 구분
· ( ㉢ ) IDS는 네트워크 상의 모든 패킷을 캡처한 후 분석하여 탐지
① 이상탐지 / 오용탐지 / 네트워크 기반
② 이상탐지 / 오용탐지 / 호스트 기반
③ 오용탐지 / 이상탐지 / 네트워크 기반
④ 오용탐지 / 이상탐지 / 호스트 기반
정답 ① — 2016 국가직과 동일 유형이지만 ①②순서가 반대로 바뀜. 정상 활동DB 비교=이상탐지. 공격패턴·시그니처DB 비교=오용탐지. 패킷 전체 캡처=NIDS.
📄 2020 경찰공무원 · 정보보호론 문20 — IDS 설명 오답★★★
침입탐지시스템(Intrusion Detection System)에서 가장 적절하지 않은 것은?
① 합법적 사용자를 침입자로 판단하는 긍정오류(False Positive)가 발생할 수 있다.
② 침입자를 합법적 사용자로 판단하는 부정오류(False Negative)가 발생할 수 있다.
③ IDS는 데이터 수집원에 따라 네트워크 기반과 호스트 기반 등으로 분류한다.
④ IDS는 패킷을 차단하며, DMZ(DeMilitarized Zone)에서만 설치가 가능하다.
정답 ④ — IDS는 패킷을 차단하지 않는다(탐지만). DMZ에서만 설치 가능하지 않다. 방화벽 앞/뒤, 내부망, 백본 등 다양한 위치에 설치 가능.
📄 2021 국가직 9급 · 네트워크보안 문18 — HIDS 오답★★★
IDS(Intrusion Detection System)에 대한 설명으로 옳지 않은 것은?
① IDS는 네트워크기반 및 호스트기반의 침입탐지시스템으로 분류할 수 있다.
② 침입탐지 기법 중 오용탐지기법은 시그니처기반 탐지 또는 지식기반 탐지로 불린다.
③ 네트워크기반 침입탐지시스템은 네트워크 전반에 대한 감시를 할 수 있다.
④ 호스트기반 침입탐지시스템은 시스템의 로그에 대한 탐지보다는 네트워크 패킷을 수집하여 공격을 판단한다.
정답 ④ — HIDS는 시스템 로그, 감사 기록, 자원 사용을 분석. 네트워크 패킷 수집은 NIDS의 특성. HIDS와 NIDS의 정보 출처가 반대로 서술된 전형적 함정.
📄 2022 국가직 7급 · 정보보호론 문17 — 이상탐지 오탐률★★★
IDS(Intrusion Detection System)에 대한 설명으로 옳지 않은 것은?
① 비정상행위(Anomaly) 침입탐지는 오용(Misuse) 침입탐지 대비 낮은 오탐률을 보이지만 알려지지 않은 새로운 공격에 취약하다.
② 호스트 기반 IDS는 트로이목마, 백도어, 내부자에 의한 공격을 탐지할 수 있다.
③ 네트워크 기반 IDS를 방화벽 외부에 설치할 경우, 내부에 설치할 때보다 더 많은 네트워크 공격을 탐지할 수 있지만 처리할 데이터가 더 많다.
④ IDS 설계 시 침입자의 행동을 엄격하게 정의할수록 공격탐지의 부정오류(False Negative)가 증가한다.
정답 ① — 이상탐지(Anomaly)는 오용탐지보다 오탐률(FP) 높음. "낮은 오탐률"은 오용탐지의 특성. 또한 새로운 공격 탐지는 이상탐지의 장점이지 취약점이 아님. 두 가지 내용이 모두 반대로 서술됨.
📄 2023 군무원 9급 · 네트워크보안 — IDS 지식기반 탐지★★
다음 IDS의 탐지 유형 중 지식기반 탐지 기법에 해당하지 않는 것은?
① 전문가 시스템 (Expert System)
② 시그니처 분석 (Signature-based Detection)
③ 페트리넷 (Petri-nets)
④ 통계적 분석
정답 ④ — 지식 기반 탐지 = 오용탐지. 오용탐지 기법: 전문가 시스템, 시그니처 분석, 패턴 매칭, 페트리넷, 상태 천이 분석. 통계적 분석은 이상 탐지(Anomaly Detection)의 기법이다.
